# 사막의 무명신 — 시드 + 설계 결정

## 시놉시스 (사용자 제공)

고대 이집트를 닮은 사막 왕국. 신탁을 받은 아이들만이 성전으로 들어가
축복을 받고, 나머지는 이름 없는 존재로 버려진다. 주인공은 누구보다
강한 영적 감응을 지녔지만, 의식의 날 아무 신도 그를 선택하지 않는다.
왕국은 이를 불길한 징조로 여기고 혼란에 빠지고, 성전의 사제들은
주인공을 감추려 한다. 그러나 주인공에게는 선택받은 자들이 듣지
못하는 목소리, 오래전 봉인된 ‘잊힌 신’의 속삭임이 들린다.

왕국이 믿어온 신들의 질서가 사실은 거대한 거짓 위에 세워졌음을 알게
된 주인공은, 자신을 배척한 세상과 무너져가는 성전 사이에서 선택을
강요받는다. 전쟁과 반란이 번지는 가운데 그는 축복받지 못한 자들의
편에 서서, 신의 힘 없이도 세계를 바꿀 수 있는지 증명하려 한다.
하지만 잊힌 신이 원하는 것은 구원이 아니라 재림일 수도 있다.
주인공은 자신이 구원자가 될지, 재앙의 문을 여는 열쇠가 될지 모른 채
사막 한가운데로 걸어 들어간다.

## 4요건 충족 점검

| 요건 | 시드의 응답 |
|---|---|
| World-differentiator | 이집트풍 신권 사회 + 모든 사람이 신탁을 받는 시스템에서 단 한 명만 받지 못함 |
| Central tension | 도용된 신들의 질서 vs 봉인된 태초신의 복수 + 그 사이에 끼인 0 신탁자 |
| Cost / constraint | 신탁 = 사회 정체성 자체. 받지 못한 자는 이름 없는 존재로 버려짐. 잊힌 신과의 동맹 = 동맹자도 결국 사라짐. |
| Sensory hook | 사막의 모래·태양·신전 의식·표식이 새겨지는 순간의 감각·잊힌 신의 속삭임 |

## 설계 결정 종합

| 항목 | 결정 |
|---|---|
| 제목 가안 | 사막의 무명신 |
| 분량 | 200회 이상 × 5,000-5,200자 (≈ 100만 자) |
| 형식 | 한국 웹소설 회차 (회당 cliffhanger·진전) |
| 언어 | 한국어 |
| POV | 3인칭 한정 (주인공 메인) + 가끔 다른 인물 회차 단위 교체 |
| 톤 | mythic epic 70% + post-divine 30%. 웹소설 분량 형식만 차용. |
| 결말 | **비공개 — 작가 전용** (`MYSTERY.md` 만 보유). 외부 공유용 시놉·로그라인에 사전 노출 금지. Arc 3 후반 부분 누설 + Arc 4 완성으로 본문 안에서 자연 노출. |
| 주인공 | 13-15세 남성, 고아·하층민, 사회 유일의 0 신탁자, 옛 태초신 신탁 수령자들의 후손. 이름 가안 "하셈". |
| 잊힌 신 | 태초의 신. 12 도용 신이 그를 봉인하고 권능을 12 영역으로 도용. 신탁 시스템 자체가 그의 도용된 권능. |
| 주인공 위치 | 잊힌 신의 의식적 동맹자 |
| 세계관 | 가상 사막 왕국 (이집트 모티프만 차용, 이름·지명·신명은 모두 가상) |
| 신의 수 | 12 (도용 신) + 1 (잊힌 태초신) |
| 정치 구조 | 신왕 1명(정점) + 신후 11명(각 신의 영역 통치자) + 12신 사제단 + 신탁 귀족 + 평민·노예 |
| 캐릭터 모델 | E (단계적 결합) + D 격자 (세대 × 진영) |
| 외부자 군 | A+B+C+D+E 다층 |
| 운영 단위 | 4 아크 거시 outline + Arc 1 (1-50회) 정밀 spec |
| 웹툰화 | 60-80화 추정 |

## 운영 메모

- 이 spec 은 첫 방문 — Arc 1 만 정밀, Arc 2-4 는 거시 outline.
- Arc 1 종료 시(약 50회 작성 후) 평가 → keep/discard → Arc 2 spec 작성.
- 한국 웹소설 ANTI-SLOP 사전(`ANTI-SLOP-KO.md`) 별도 작성: 사극 클리셰·먼치킨 표현·과잉 형용사·반복 수사를 회피.
- voice 단계에서 sensory hook 의무화 (웹툰화 의식: 매 회차에 시각적으로 살릴 장면 1개 이상).
- 주인공 외 다른 인물의 시점은 5-10회마다 한 번 (회차 단위 교체). 누구의 시점인지 회차 첫머리에 명시.
- **결말 노출 룰**: 사용자 제공 시놉의 결말 모호성("구원자가 될지, 재앙의 문을 여는 열쇠가 될지 모른 채") 을 외부 공유 단계까지 보존. 작가용 결말 결정은 `MYSTERY.md` 만 보유. 본문 노출 전략은 `MYSTERY.md` 의 "Reveal Strategy" 표 참조.
